Principais Tags

<'form>

Exemplo: Ao criar um formulário e colocar seus campos de input dentro dele, todos estarão relacionados de alguma forma. Ao clicar no botão input com o atributo "submit", ele automaticamente envia o conteúdo dos outros campos input dentro da tag Form. O atributo "name" do input text, serve para especificar o modo com que o servidor está esperando receber os dados, pra saber o que é cada um, é como se você passase um dado é um parâmetro.

No caso do google, para fazer uma pesquisa, no url ele recebe algo tipo "q=texto_passado_no_input". Com isso, complet a url de pesquisa, ficando "google.com/search?q=texto_passad_no_input".

O atributo "action" da tag form, serve para indicar qual página ou servidor irá receber os dados do formulário. o método "get" está relacionado ao meio de envio, e existe o método "post", além do "get". Nesse exemplo, estamos enviando os dados para o servidor do google.

Semântica em Formulários

Trabalhando legendas em campos de formulários, assim como legendamos imagens. É importante por conta da acessibilidade. Usamos a tag <'label> para colocar a legenda. O atributo "for" do label serve para indicar qual o campo de input para qual aquela legenda descreve. Precisamos unir esse "for" com o input certo, fazemos isso por meio do "id", e ambos (for e id) recebem o mesmo nome.

Organizando Entrada de Dados

A tag <'fieldset> serve para colocar um grupo de input dentro de uma caixinha. É bastante útil para separar os formulários em blocos. E para cada bloco fieldset, temos uma legenda que é colocada por meio da tag <'legend>.
