Wireframe: São esçobos simples de telas de produtos digitais, como sites e aplicativos.

**Onde tem parênteses, são < >

  • (div id="") (/div)
  • (header) (/header) : header é o cabeçado da seção, dá para colocar um menu, um logotipo, formulário de busca, menu de navegação, etc.
  • (nav) (/nav) : navegação principal da página, nem todo link precisa tá na tag nav. O normal é colocá-la dentro das tags padrão na página (tipo o header), as que não mudam. Normalmente temos um nav dentro do header, por conta de menus e tal.
  • (main) (/main) : exclusiva para o conteúdo principal da página, nada além. temos que ser semânticos.
  • (footer) (/footer) : o rodapé da página. dá para por um footer em cada trecho de artigo da página.

Outras tags

  • (section)(/section) : Representa uma seção da página, não tem tanto valor semântico, é mais para dividir o conteúdo da página em blocos, separar um conteúdo relevante da página.
  • (article)(/article) : É a tag principal do documento. É mais comumente usado para conter informações que podem ser distribuídas independentemente do restante do site ou aplicativo em que aparecem. Tipo aqueles títulos de máterias nas prévias de link.
  • (aside)(/aside) : Representa uma seção de uma página que consiste de conteúdo que é tangencialmente relacionado ao conteúdo do seu entorno. Tipo um menu lateral para diferentes páginas do site. Pode ser deixada também para funcionar como um banner de propagandas.

Tags Não Semânticas

  • (div)(/div) : Serve para criar blocos, que nos ajudam em questões visuais.
  • (span)(/span) : Serve para dar um destaque à uma determinada parte do texto.
