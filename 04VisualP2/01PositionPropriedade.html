Propriedade Position

A propriedade position especifica o tipo de método de posicionamento usado para um elemento ( static, relative, fixed, absolute ou sticky )

Static

Os elementos HTML são posicionados desse jeito por padrão. Esse é o comum. Ele não é posicionado de nenhuma maneira especial, ele é posicionado de acordo com o fluxo padrão da página.
div .static { position: static ; border: 3px solid #73AD21 ;}

Esse aqui é o tal do position: sticky. NÃO TEM NADA A VER COM O POSITION: STATIC

Relative

Um elemento com position: relative é posicionado em relação à sua posição normal. Sem contar que ele continua ocupando o mesmo espaço na página, como se nem tivesse saído do lugar. Fica algo no estilo de layers, onde a página está ao fundo e o elemento posicionado relative fica por cima.
Definir as propriedades top, right, bottom, left de um elemento relativamente posicionado fará com que ele seja ajustado para longe de sua posição normal. Os outros conteúdos não serão alterados por isso, pois o resto da página continua como se ele nunca tivesse saído de sua posição.
div .relative { position: relative ; left: 30px ; border: 3px solid #73AD21 ;}
Apesar de estar bem à direita, o elemento está bem aqui acima e sem toda essa indentação. Com essa propriedade, dá para colocar essa div em qualquer canto da tela.

Fixed

Um elemento com position: fixed, está posicioniado em relação à janela de visualização, mesmo se a página for rolada. (Tipo aqueles botões que ficam fixados no cantinho da tela e serve pra voltar ao todo, apesar de ir descendo na página, ele continua no mesma posição da tela.). As propriedades top, right, bottom, left são usadas para posicionar o elemento.
div .fixed { position: fixed ; bottom: 0 ; right: 0 ; width: 300px ; border: 3px solid #73AD21 ;} OLHA EU AQUI, FIXED
O componente está no canto inferior direito. Um elemento fixo não deixa uma lacuna na página onde normalmente estaria localizado. Como é o caso do relative.

Absolute

Um elemento com position: absolute está posicionado em relação ao seu "pai" mais próximo, ao invés da posicioná-lo em relação à janela de visualização (tipo o fixed). De qualquer forma, se um elemento posicionado em absoluto não tiver um "pai" posicionado, ele usará o corpo do documento e se moverá junto com a rolagem da página.
Elementos posicionados em absoluto são removidos do fluxo normal e podem sobrepor elementos da página.
Exemplo:
div.relative { position: relative; width: 400px; height: 200px; border: 3px solid #73AD21 ;}
div.absolute { position: absolute; top: 80px; right: 0; width: 210px; height: 100px; border: 3px solid #73AD21 ;}

Sticky

Um elemento com position: sticky está posiciodado com base na rolagem do usuário (tipo aqueles header com menus de páginas que seguem no topo até o fim da página). Um elemento sticky alterna entre relative e fixed, dependendo da posição de rolagem. Ele é posicionado em relative até que uma determinada posição de deslocamento seja encontrada na janela de exibição. Então ele se fixa no lugar (tipo o position: fixed).
Nesse exemplo foi passado como parâmetro TOP: 0; mas também pode ser passado outros parâmetros
DEVE-SE especificar pelo menos uma dimensão TOP, RIGHT, BOTTOM, LEFT, em relação a viewport para que o sticky possa ficar localizado na tela.

Z-Index: Overlapping Elements (Elementos Sobrepostos)

Quando elementos estão posicionados, eles podem sobrepôr outros elementos. A propriedade z-index especifica a ordem da pilha de um elemento (qual elemento deve ser colocado na frente ou atrás dos outros).
Um elemento pode receber um valor de ordem positivo ou negativo, -1 significa que o elemento vai ficar atrás, 1 significa que o elemento vai ficar na frente.
SE DOIS ELEMENTOS POSICIONADOS SE SOBREPÕEMSEM UM Z-INDEX ESPECIFICADO, O ELEMENTO POSICIONADO POR ÚLTIMO NO CÓDIGO HTML SERÁ MOSTRADO NA PARTE SUPERIOR!!!
