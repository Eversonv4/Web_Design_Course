div:before {
        content: "";
      }

      div:after {
        content: "";
      }

Sempre que for usar essas pseudo-classes, tem que usar o content, mesmo que seja vazio. tem que ter.

Esse texto inserido por css dentro do content não pode ser selecionado e nem é usado pelos mecanismos de busca. Então esse conteúdo before ou after, deve ser usado de preferência para conteúdos visuais ou estilos. É o que faz mais sentido.

Por mais que não seja colocado nada dentro do content, existe algo lá e esse elemento é do tipo inline, ou seja, são elementos do tipo linha, como as tags <'b>, <'em>, <'i>, etc.
